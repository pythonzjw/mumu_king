# CONTEXT_HANDOFF

## 当前目标
修复日常漏执行，优化技能关键字解析可见性，并降低主循环 CPU/磁盘占用。

## 已完成
- 日常调度对活动、限时活动、七日狂欢增加固定坐标兜底入口，减少 HOME 红点模板漏识别。
- 装备日常在体力等待窗口内固定执行一次，不再只依赖装备红点。
- 活动/七日领取改为 OCR 多轮点击：每次点一个后重新截图，空结果复扫 2 次后退出。
- 限时活动「签到」改为包含匹配，并逐行排除已签到/未开启/继续领取；下滑后等 1s 再识别。
- 七日狂欢不再只处理有红点天数，进入后遍历第 1～7 天，挑战/好礼都做多轮领取/免费扫描。
- 技能优先级/禁选关键字支持中英文逗号、顿号、分号、换行分隔；匹配时忽略 OCR 空白。
- Worker 启动打印实际优先级/禁选列表，技能选择时打印禁选过滤后的可选卡。
- GUI 默认关闭调试截图；战斗等待从 2s 调到 4s；广告 OCR 兜底改为连续 UNKNOWN 后才触发。

## 关键决策
- 体力等待窗口内优先“不漏点”，接受日常多花几十秒。
- 装备一键升级/合成无红点时也执行一次，视为安全无副作用。
- 七日狂欢遍历 1～7 天，避免小红点识别失败导致整天跳过。
- CPU 优化优先低风险降频，不改核心状态机；需要截图排查时手动勾选 debug。
- 不新增依赖，不改状态机结构。
